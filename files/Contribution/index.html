    Comments

    • The scope of a Contribution instance includes only the actions of a single agent in contributing to a single artifact, which may be performed in one continuous effort, or several discontinuous sessions of work. The agent can be an individual person, a software agent, or a group or organization of individuals acting together. Contribution instances are used to organize information about when, where, how (e.g. what roles were played), and on whose behalf these actions were performed.

      a primary author, editor, figure designer, data generator, etc. . Values of
      this property may be framed as activities or as contribution roles (e.g. using

    comments:

      \ be captured in a single Contribution object. However, if data creators want\
      \ to provide metadata specific to each activity (e.g. when or how each was performed),\
      \ then a separate Contribution object should be created for each activity."
